<CaseStudyGrid>

<CaseStudySection title="Context">

A personal portfolio built with Next.js, TypeScript, and Velite for type-safe MDX content. I designed and developed the entire site as a demonstration of my engineering practices — fast, accessible, and content-driven.

</CaseStudySection>

<CaseStudySection title="Challenge">

I needed a single place to present my projects, experience, and contact information to recruiters — with enough depth to show how I think about building software, without sacrificing scanability for busy hiring managers.

</CaseStudySection>

<CaseStudySection title="Approach">

The site uses a single-page layout for quick scanning, dedicated MDX case study pages for deeper project write-ups, and a content layer that validates frontmatter at build time. Dark mode, semantic HTML, and subtle scroll animations keep the experience polished without sacrificing performance. The Velite schema enforces required fields so broken content never ships.

</CaseStudySection>

<CaseStudySection title="Impact" colSpan={12}>

- Content updates require only adding an MDX file and pushing to GitHub
- Type-safe project schema catches missing fields at build time
- Deployed on Vercel with automatic previews on every push

</CaseStudySection>
